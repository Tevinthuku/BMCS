The Normalization Process

Its just a process of organising data in the database.


Purpose Of Normalization

  • Eliminate Useless data
  • Ensure data dependency make sense, makes sure they are logically stored.


Problemns That occur without Normalization



  • Insertion Anomalies
  • Updating Anomalies
  • Deletion Anomalies

These anomalies are very easy to understand


Look at it this way..

1. On Insertion Anomalies

We will have problems when we want to insert data into the database

2. On Update Anomalies

We will have problems when we want to update data in the database

2. On Deletion Anomalies

We will have problems when we want to delete data from the database

What are this problems exactly ?

1. On Insertion Anomalies

Say for example we have a new student that has been registered in an institution and has not nesessarilly chosen the course of his choice, We will need to insert NULL, on the course field and this will bring problems if the field does not allow NULL Values. Think of a new banker that has not been posted to a branch, or a new security guard that has been registered but has not been given a post.


2. On Update Anomalies

This occurs if we have data thats in more than one place, if we update one part of the data, then the duplicate data will be inconsistent with the updated data, this would mean that we need to update the data in more than one place, if we are to maintain consistency or data.


2. On Deletion Anomalies

This could go in two awkward ways,

  • If we delete data on one row, the entire record is deleted.
  • On deleting a record, we can have part of the record still in the database because of unawareness , that the data is saved elsewhere.

For instance, when one department closes, then it would mean that the Info of employees working in that department gets lost as well.





Normalization Rules

Ill Briefly outline the rules that are followed for any Normalization process upto the 3rd normal form.



1. First Normal Form. (1NF)

  • A collumn cannot hold multiple values, just one atomic value. Each table attribute must have a single atomic value.
    • Name Age Phone Number
    Dennis 20 07234323, 0734243
    Bob 30 07234323


    Will become::



    Name Age Phone Number
    Dennis 20 07234323
    Dennis 20 0734243
    Bob 30 07234323


    2NF

  • Table is in the 1NF
  • Remove any partial dependency

    • This means that all non prime attribute must be fully functionally dependent on a prime key attribute


    A non prime Attribute ==> This is an attribute that isnt part of any candidate key.

    3NF



    • Must be in 2NF
    • No non prime attribute shd be dependent on another non prime attribute

    This is what we mean when we say, we must eliminate transitive dependency.
